# Cathode-Ray-Tube-Amusement-Device-Simulator-1947---Hybrid-Recreation
1947年の最古の電子ゲーム『Cathode-Ray Tube Amusement Device』の根幹メカニクス（放物線弾道や時限爆発）を再現しつつ、最大1000機のターゲット配置、自律巡航移動、タイムアタックモード、Web Audio APIによるシンセシス音響など、現代的なアレンジを融合させた現代発展型Webシミュレーターです。

本プロジェクトは、特許資料に記載された「放物線弾道（磁場偏向）」および「時間指定によるデフォーカス爆発判定」という根幹の物理演算・メカニクスをWebブラウザ（HTML5 Canvas/JavaScript）上に忠実にエミュレートしつつ、現代的なゲームプレイの快適性と検証性を高めるための動的アレンジ要素を融合させた「ハイブリッド・リクリエーション」として開発されました。外部ライブラリに一切依存せず、単一のHTMLファイル（バニラJavaScript）のみで完全完結します。

操作体系
発射ボタン / Spaceキー: 自機位置から電子ビーム（光点）を射出します。

Trajectory (軌道): ダイヤルで発射角度（0°〜90°、直上基準から右方向）を調整します。

Blast Time (爆発): 光点が発射されてからデフォーカス（拡散）して爆発するまでの時間を調整します。

Spot Size (大きさ): 爆発発生時における、光点の広がり（＝命中判定の半径）を調整します。

📋 仕様の明確な境界（再現とアレンジ）
意図しない仕様の混同を防ぐため、特許に基づく再現部分と、現代的な拡張部分を論理的に切り分けて設計しています。

1. 特許準拠の再現要素（Original Specs）
アナログパラメータの完全連動: 角度調整ノブ（Trajectory）および爆発タイマー（Blast Time）によって光点の弾道を制御する、実機の操作系をそのままシミュレートしています。

磁場偏向の物理弾道: ビーム軌道は直線ではなく、陰極線管の偏向コイルによる影響（または重力加速度）をエミュレートした、滑らかな「放物線（カーブ）」を描いて飛びます。

時限デフォーカス判定: 移動中の光点自体には当たり判定がなく、指定した「Blast Time」に到達した瞬間にビームが拡散し、その範囲（Spot Size）の中に標的が含まれている場合のみ命中とみなす、本来の時限爆発ロジックを厳密に再現しています。

2. 現代的アレンジ要素（Modern Adjustments）
動的アセット描画: オリジナルではセロハン等の透明オーバーレイを画面に直接貼り付けていた標的および自機位置を、 Canvas上に動的なグラフィックとして電子描画しています。標的のデザインは当時の図面に基づいた戦闘機および潜水艦のシルエットを採用しました。

Target Count（最大1000機拡張）: 負荷検証およびゲーム性拡張のため、敵機の数を1機から最大1000機までリアルタイムに増減可能なパラメータを追加しました。

Moving Targets（自律巡航運動）: 固定されていた敵機が、レーダーの円形境界内を自動で漂い、外周に達した際に幾何学的に反転バウンドする移動ロジックを搭載しています。

タイムアタック型状態遷移: 初弾発射から全機撃破までのタイムを performance.now() によるミリ秒精度で計測する、Status Board（計器盤）を追加しました。撃破されたターゲットは消滅し、全滅して「CLEAR!」となった4秒後に自動で完全再配置（リセット）されます。

Web Audio APIによるシンセシス音響: オシレーターの周波数・ゲインの対数減衰パラメータをプログラム制御し、発射・爆発・撃墜・クリア時のアナログ効果音を完全内部生成しています。

🛠 技術的アプローチと最適化の論理
1. 高解像度ディスプレイ（DPR）環境における座標・描画の一致
Retinaディスプレイ等の高画素密度環境下で発生する、Canvas要素の表示位置と内部計算座標の乖離を排除しました。一括的な拡大命令（setTransform）を廃止し、内部の幾何学演算（座標や衝突距離判定）は物理ピクセルベースの絶対座標系で一元管理。グラフィックをレンダリングする個別のタイミングでのみ dpr（デバイスピクセル比）をダイレクトに乗算する設計としました。これにより、あらゆるディスプレイ解像度において画素単位で正確な同期を保証しています。

2. 画面サイズに追従する弾道ベクトルの動的スケーリング
可変する円形レーダーの半径（radarRadius）に基づき、光点の初速（speed）および下方向への偏向係数（gravity）を動的に決定する以下のスケーリング方程式を実装しました。

speed = (radarRadius * 2.2) / 150

gravity = radarRadius * 0.00005
これにより、最大Blast Time設定値（150フレーム）の時点で正確にスクリーンの対角線の最奥（直径の約1.1倍の物理距離）へ弾道が到達するよう運動方程式を調律。画面のアスペクト比や絶対解像度に依存せず、全域へ爆発を届かせるアナログ調整が破綻なく機能します。

3. 最大1000機配置時の空間排他（重なり回避）と計算量 O(N 
2
 ) の制御
大量の敵機オブジェクトを動的生成する際、機体同士が重複して重なり合う問題を防ぐため、「距離減衰型の空間排他配置ロジック」を実装しています。新規生成座標から既存の機体群との距離を Math.hypot で計測し、当たり判定半径の2.5倍以上の離隔距離を保つ配置を走査します。1000機配置時のような高密度状況下での無限ループ（ブラウザのハングアップ）を回避するため、試行上限（アテンプト限界）を50回に制限する調停ロジックを導入し、軽量な実行速度と独立したオブジェクト配置を両立させました。

4. オーディオミキシングにおける「引き算」の最適化
Web Audio APIによるシンセサイザー音響を実装するにあたり、オブジェクトの自律運動によって大量に発生する「画面端でのバウンド音」は、高密度巡航時に毎秒数百回以上の同時発音（ボイス数超過）を引き起こし、デジタル・クリッピング（波形の飽和による音割れ）や音声スレッドのハングアップを招くリスクがありました。これを防ぐため、バウンド時の環境音をあえて設計段階でオミットし、プレイヤーの能動的操作（発射、爆発、命中、クリア）に直結するシグナルのみを明瞭に出力するオーディオデザインへと最適化しています。
